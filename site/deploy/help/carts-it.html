I diversi modi in cui le cartucce possono essere caricate in JS7800 sono dettagliati nel resto di questa scheda. JS7800 dovrebbe essere compatibile con la stragrande maggioranza delle ROM di cartucce disponibili per Atari 7800 (sia versioni retail originali che homebrew).

Le ROM di cartucce Atari 7800 hanno tipicamente estensione file .bin o .a78. L'estensione .a78 è preferita poiché contiene informazioni di intestazione che forniscono all'emulatore informazioni su come la cartuccia particolare dovrebbe essere interpretata. Vale anche la pena notare che JS7800 supporta ROM di cartucce contenute in file .zip.

Menu a Discesa Cartucce

Il menu "a Discesa Cartucce" (evidenziato in rosso sotto) contiene una lista di cartucce Atari 7800 da selezionare.

Seleziona Cartuccia Atari 7800...

Per default, la lista cartucce è popolata con una selezione di eccellenti titoli homebrew. Per caricare una cartuccia, seleziona semplicemente l'elemento appropriato nel menu.

Liste cartucce personalizzate sono anche supportate. Per più informazioni su creazione e caricamento di liste cartucce personalizzate, fai riferimento alla pagina GitHub Liste Cartucce.

Seleziona File Locali

Il pulsante "Seleziona File Locali" (evidenziato in rosso sotto), è usato per selezionare e caricare un file ROM cartuccia Atari 7800 che si trova sul dispositivo locale.

Seleziona Cartuccia Atari 7800...

Una volta premuto il pulsante, un selettore file sarà mostrato che permette la selezione di un file locale. Per default, il selettore file limita i file mostrati a estensioni .a78, .bin, .zip, e .json.

Il selettore file locale può anche essere usato per selezionare un file "lista cartucce" (estensione file .json). Selezionare un file "lista cartucce" aggiornerà il contenuto del Menu a Discesa Cartucce. Per più informazioni su liste cartucce, fai riferimento alla pagina GitHub Liste Cartucce.

Seleziona File Remoti

Il pulsante "Seleziona File Remoti" (evidenziato in rosso sotto), è usato per specificare l'URL di una ROM cartuccia Atari 7800 remota.

Seleziona Cartuccia Atari 7800...

Una volta premuto il pulsante, una finestra di input sarà mostrata che permette di specificare l'URL a un file ROM remoto. Per esempio, inserire la seguente URL causerà il caricamento del gioco homebrew, "bonQ":

http://raz0red.github.io/js7800/roms/bonq.a78

Il selettore file remoto può anche essere usato per selezionare un file "lista cartucce" remoto (estensione file .json). Selezionare un file "lista cartucce" aggiornerà il contenuto del Menu a Discesa Cartucce. Per più informazioni su liste cartucce, fai riferimento alla pagina GitHub Liste Cartucce.

Trascina e Rilascia File

Trascinare e rilasciare ROM cartucce Atari 7800 locali può essere usato come alternativa a selezionare file tramite il browser file locale (vedi Seleziona File Locali).

Come dimostrato sopra, file ROM locali possono essere caricati semplicemente trascinandoli e rilasciandoli sull'emulatore da Esplora File su Windows o Finder su Mac.

In aggiunta a file ROM cartucce, file "lista cartucce" (estensione file .json) possono anche essere trascinati e rilasciati sull'emulatore. Trascinare e rilasciare un file "lista cartucce" causerà l'aggiornamento del contenuto del Menu a Discesa Cartucce. Per più informazioni su liste cartucce, fai riferimento alla pagina GitHub Liste Cartucce.

Trascina e Rilascia Link

Trascinare e rilasciare link che puntano a ROM cartucce Atari 7800 remote può essere usato come alternativa a specificare URL tramite il selettore file remoto (vedi Seleziona File Remoti).

Come dimostrato sopra, ROM possono essere caricate semplicemente trascinando e rilasciando link file remoti sull'emulatore. Nell'esempio specifico sopra, un link è trascinato dal Forum Programmazione Atari 7800 di AtariAge, un'ottima fonte per titoli homebrew in sviluppo e completati. Vale anche la pena notare che alcuni link sul forum AtariAge richiedono che il trascinamento sia eseguito tramite il link "conteggio download" versus il link del titolo del gioco (come mostrato sopra).

In aggiunta a link file ROM cartucce, link file "lista cartucce" possono anche essere trascinati e rilasciati sull'emulatore. Trascinare e rilasciare un link file "lista cartucce" causerà l'aggiornamento del contenuto del Menu a Discesa Cartucce. Per più informazioni su liste cartucce, fai riferimento alla pagina GitHub Liste Cartucce.

Parametri Richiesta

JS7800 supporta il seguente insieme di parametri richiesta. I parametri richiesta possono essere aggiunti all'URL di JS7800 per controllare varie opzioni dell'emulatore. Per esempio, il parametro richiesta cart può essere usato per forzare il caricamento di una cartuccia specifica quando JS7800 è inizialmente mostrato.

cart
Il parametro richiesta cart può essere usato per caricare una cartuccia specifica quando la pagina emulatore è inizialmente caricata. La seguente URL causerà il caricamento del gioco homebrew, "bonQ":

https://raz0red.github.io/js7800/?cart=http://raz0red.github.io/js7800/roms/bonq.a78

cartlist
Il parametro richiesta cartlist può essere usato per caricare una lista cartucce nel Menu a Discesa Cartucce quando JS7800 è inizialmente caricato. La seguente URL causerà il popolamento di una lista demo homebrew nel menu a discesa cartucce:

https://raz0red.github.io/js7800/?cartlist=http://raz0red.github.io/js7800/roms/romlist-homebrew-demos.json

Per più informazioni su liste cartucce, fai riferimento alla pagina GitHub Liste Cartucce.

Parametri richiesta multipli possono essere combinati per impostare più di un'opzione quando JS7800 è caricato. La seguente URL caricherà un gioco specifico, "bonQ", e una lista cartucce contenente demo homebrew:

https://raz0red.github.io/js7800/?cart=http://raz0red.github.io/js7800/roms/bonq.a78&cartlist=http://raz0red.github.io/js7800/roms/romlist-homebrew-demos.json

Dato che aggiungere parametri richiesta tipicamente risulterà in URL lunghi, l'uso del servizio accorciamento URL di GitHub, git.io, è raccomandato. La seguente URL è la versione accorciata dell'esempio sopra, contenente entrambi parametri richiesta cart e cartlist:

https://git.io/JfcmO
