As diferentes formas como cartuchos podem ser carregados no JS7800 são detalhadas ao longo do resto desta aba. JS7800 deve ser compatível com a vasta maioria das ROMs de cartuchos disponíveis para Atari 7800 (tanto lançamentos originais quanto homebrew).

ROMs de cartuchos Atari 7800 tipicamente têm extensão de arquivo .bin ou .a78. A extensão .a78 é preferida pois contém informações de cabeçalho que fornecem ao emulador informações sobre como o cartucho particular deve ser interpretado. Também vale notar que JS7800 suporta ROMs de cartuchos contidas em arquivos .zip.

Menu Suspenso de Cartuchos

O menu "Suspenso de Cartuchos" (destacado em vermelho abaixo) contém uma lista de cartuchos Atari 7800 para selecionar.

Selecionar Cartucho Atari 7800...

Por padrão, a lista de cartuchos é populada com uma seleção de excelentes títulos homebrew. Para carregar um cartucho, simplesmente selecione o item apropriado no menu.

Listas de cartuchos personalizadas também são suportadas. Para mais informações sobre criação e carregamento de listas de cartuchos personalizadas, consulte a página GitHub de Listas de Cartuchos.

Selecionar Arquivos Locais

O botão "Selecionar Arquivo Local" (destacado em vermelho abaixo), é usado para selecionar e carregar um arquivo ROM de cartucho Atari 7800 que está localizado no dispositivo local.

Selecionar Cartucho Atari 7800...

Uma vez que o botão é pressionado, um seletor de arquivos será mostrado que permite a seleção de um arquivo local. Por padrão, o seletor de arquivos limita os arquivos mostrados a extensões .a78, .bin, .zip, e .json.

O seletor de arquivos local também pode ser usado para selecionar um arquivo "lista de cartuchos" (extensão de arquivo .json). Selecionar um arquivo "lista de cartuchos" atualizará o conteúdo do Menu Suspenso de Cartuchos. Para mais informações sobre listas de cartuchos, consulte a página GitHub de Listas de Cartuchos.

Selecionar Arquivos Remotos

O botão "Selecionar Arquivos Remotos" (destacado em vermelho abaixo), é usado para especificar a URL de uma ROM de cartucho Atari 7800 remota.

Selecionar Cartucho Atari 7800...

Uma vez que o botão é pressionado, um diálogo de entrada será mostrado que permite especificar a URL para um arquivo ROM remoto. Por exemplo, inserir a seguinte URL causará o carregamento do jogo homebrew, "bonQ":

http://raz0red.github.io/js7800/roms/bonq.a78

O seletor de arquivos remoto também pode ser usado para selecionar um arquivo "lista de cartuchos" remoto (extensão de arquivo .json). Selecionar um arquivo "lista de cartuchos" atualizará o conteúdo do Menu Suspenso de Cartuchos. Para mais informações sobre listas de cartuchos, consulte a página GitHub de Listas de Cartuchos.

Arrastar e Soltar Arquivos

Arrastar e soltar ROMs de cartuchos Atari 7800 locais pode ser usado como alternativa a selecionar arquivos via o navegador de arquivos local (veja Selecionar Arquivos Locais).

Como demonstrado acima, arquivos ROM locais podem ser carregados simplesmente arrastando e soltando no emulador do Explorador de Arquivos no Windows ou Finder no Mac.

Em adição a arquivos ROM de cartuchos, arquivos "lista de cartuchos" (extensão de arquivo .json) também podem ser arrastados e soltados no emulador. Arrastar e soltar um arquivo "lista de cartuchos" causará a atualização do conteúdo do Menu Suspenso de Cartuchos. Para mais informações sobre listas de cartuchos, consulte a página GitHub de Listas de Cartuchos.

Arrastar e Soltar Links

Arrastar e soltar links que apontam para ROMs de cartuchos Atari 7800 remotas pode ser usado como alternativa a especificar URLs via o seletor de arquivos remoto (veja Selecionar Arquivos Remotos).

Como demonstrado acima, ROMs podem ser carregadas simplesmente arrastando e soltando links de arquivos remotos no emulador. No exemplo específico acima, um link está sendo arrastado do Fórum de Programação Atari 7800 do AtariAge, uma ótima fonte para títulos homebrew em desenvolvimento e completados. Também vale notar que alguns links no fórum AtariAge requerem que o arrasto seja executado via o link "contagem de downloads" versus o link do título do jogo (como mostrado acima).

Em adição a links de arquivos ROM de cartuchos, links de arquivos "lista de cartuchos" também podem ser arrastados e soltados no emulador. Arrastar e soltar um link de arquivo "lista de cartuchos" causará a atualização do conteúdo do Menu Suspenso de Cartuchos. Para mais informações sobre listas de cartuchos, consulte a página GitHub de Listas de Cartuchos.

Parâmetros de Requisição

JS7800 suporta o seguinte conjunto de parâmetros de requisição. Parâmetros de requisição podem ser adicionados à URL do JS7800 para controlar várias opções do emulador. Por exemplo, o parâmetro de requisição cart pode ser usado para forçar o carregamento de um cartucho específico quando JS7800 é inicialmente mostrado.

cart
O parâmetro de requisição cart pode ser usado para carregar um cartucho específico quando a página do emulador é inicialmente carregada. A seguinte URL causará o carregamento do jogo homebrew, "bonQ":

https://raz0red.github.io/js7800/?cart=http://raz0red.github.io/js7800/roms/bonq.a78

cartlist
O parâmetro de requisição cartlist pode ser usado para carregar uma lista de cartuchos no Menu Suspenso de Cartuchos quando JS7800 é inicialmente carregado. A seguinte URL causará o preenchimento de uma lista de demos homebrew no menu suspenso de cartuchos:

https://raz0red.github.io/js7800/?cartlist=http://raz0red.github.io/js7800/roms/romlist-homebrew-demos.json

Para mais informações sobre listas de cartuchos, consulte a página GitHub de Listas de Cartuchos.

Parâmetros de requisição múltiplos podem ser combinados para definir mais de uma opção quando JS7800 é carregado. A seguinte URL carregará um jogo específico, "bonQ", e uma lista de cartuchos contendo demos homebrew:

https://raz0red.github.io/js7800/?cart=http://raz0red.github.io/js7800/roms/bonq.a78&cartlist=http://raz0red.github.io/js7800/roms/romlist-homebrew-demos.json

Dado que adicionar parâmetros de requisição tipicamente resultará em URLs longas, o uso do serviço de encurtamento de URL do GitHub, git.io, é recomendado. A seguinte URL é a versão encurtada do exemplo acima, contendo ambos parâmetros de requisição cart e cartlist:

https://git.io/JfcmO
